# AvatarProject
# Цель и область применения

В этом документе содержится общее представление о системе 3D Avatar Control, проекте Unity, который интегрирует возможности распознавания позы Google MediaPipe для создания скелетной анимации 3D-гуманоидного аватара в реальном времени. Система получает данные о позе через UDP от внешнего процесса MediaPipe, преобразует их в пространство координат Unity и применяет вращение костей для анимации персонажа аватара.
В этом обзоре рассматриваются следующие вопросы:

- Общая архитектура системы и поток данных
- Обязанности основных компонентов
- Используемые технологии и фреймворки
- Текущий объем реализации и ограничения

# Архитектура системы
Система управления 3D-аватаром состоит из трех основных слоев, которые работают вместе, чтобы преобразовать реальные движения человека в анимированные движения персонажа в 3D:
<img width="621" height="866" alt="image" src="https://github.com/user-attachments/assets/e0f8eb16-60a8-4472-8532-a2105417e76a" />
Архитектура следует четкому конвейеру данных:

Внешние источники данных: MediaPipe работает как отдельный процесс, определяя позу человека по видеовходу и передавая объекты по UDP PoseLandmarkerResult
Unity Application Core: получает и обрабатывает данные через и MediaPipeUdpReceiverMediaPipeAvatarController
Конвейер рендеринга: Стандартный конвейер рендеринга Unity отображает аватар с 9 различными материалами

Использован MediaPipe Framework, написанный на языке C++ и интегрированный в C# для прямого использования в unity,возможные среды выполнения Android,ios,(VR)

Рекомендации по импорту
* Важно при использования проета учетная запись должна быть на английском языке
* Использованная версия Unity 6.000.33f1
* Windows
